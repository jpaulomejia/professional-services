Comeback Ritual — Run This Every Time You Return

1
Visit the live site

Open your browser and go to your live URL. Remind yourself what exists and what it looks like.

2
Check Supabase

Log into supabase.com → Table Editor → contacts. See if anyone submitted a form while you were away.

3

Open your website folder. Scan the files. Read the last change you made. Now you know where you left off.

Phase 1 — Build It

Create your website files

  • Create a website folder
  • Build HTML pages (index, services, contact)
  • Create styles.css for design
  • Create script.js for form handling
  • Add Supabase CDN + script to contact.html
Phase 2 — Store It

Set up Supabase database

  • Create account at supabase.com
  • Create new project
  • Run SQL to create contacts table
  • Click "Run and enable RLS"
  • Run SQL to add insert policy
  • Copy Project URL + Publishable key
  • Paste into script.js
Phase 3 — Notify It

Set up Formspree emails

  • Create account at formspree.io
  • Create new form
  • Copy the Form ID from the URL
  • Paste into script.js
Phase 4 — Ship It

Push to GitHub & go live

  • Create account at github.com
  • Create new public repository
  • Run git commands in Terminal
  • Go to repo Settings → Pages
  • Set branch to main → Save
  • Wait 2 min → site is live!
Phase 5 — Update It

Make changes to the live site

  • Edit files in VS Code
  • Save with Cmd + S
  • Run the 3 git commands below
  • Wait 1–2 minutes
  • Hard refresh: Cmd + Shift + R
git add .
git commit -m "what I changed"
git push

Troubleshooting

Form submits but nothing saves
Site must be on GitHub Pages — it won't work on a local file. Push to GitHub first.
"supabase already declared" error
Rename your client variable from supabase to supabaseClient in script.js.
Changes not showing on live site
Run git push, wait 2 min, then press Cmd + Shift + R to hard refresh.
GitHub asking for password
Use your Personal Access Token, not your regular password. GitHub → Settings → Developer Settings → Tokens.

Supabase — Create Contacts Table

create table contacts (
  id         uuid default gen_random_uuid() primary key,
  first_name text,
  last_name  text,
  email      text,
  phone      text,
  address    text,
  service    text,
  message    text,
  created_at timestamp default now()
);

Supabase — Add Insert Policy

create policy "Allow public to insert contacts"
on contacts
for insert
to anon
with check (true);

Run this after creating the table. Allows the contact form to save data.
