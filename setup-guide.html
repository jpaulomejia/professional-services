Website Setup Guide

Step-by-step instructions for publishing your site and setting up your database

Professional Services Website

What This Guide Covers

This guide has three parts:
  • Part 1 — Install Git and create a GitHub account
  • Part 2 — Upload your website to GitHub and make it live
  • Part 3 — Set up Supabase to store and view contact form submissions
When you're done, your website will be publicly accessible and every contact form submission will be saved to your Supabase database.
Part 1 — Install Git & Create a GitHub Account One-time setup — you only do this once

What is Git and GitHub?

Git is a tool that tracks changes to your files. GitHub is a website where you store those files in the cloud. GitHub Pages (a free GitHub feature) then serves those files as a live website.

1 Check if Git is already installed

Open Terminal on your Mac (search "Terminal" in Spotlight with Cmd + Space).

Type the following and press Enter:

git --version

If you see something like git version 2.x.x — Git is already installed, skip to Step 3.

If you see "command not found" — continue to Step 2.

2 Install Git

In Terminal, type:

xcode-select --install

A popup will appear — click Install and wait for it to finish (takes a few minutes).

Once done, run git --version again to confirm it worked.

3 Create a free GitHub account

Go to github.com and click Sign up.

Use your email, create a username and password, and verify your account.

Your GitHub username will appear in your website URL, so pick something professional (e.g. jpaulomejia).
4 Tell Git your name and email

In Terminal, run these two lines (replace with your actual name and email):

git config --global user.name "Juan Paulo Mejia"
git config --global user.email "paulo.j.mejia@gmail.com"
Part 2 — Upload Your Website to GitHub & Make It Live Takes about 10 minutes
1 Create a new repository on GitHub

Log in to github.com and click the + icon (top right) → New repository.

  • Repository name: professional-services
  • Visibility: Public
  • Leave everything else unchecked

Click Create repository. Leave this page open — you'll need the URL on it.

2 Open Terminal and go to your website folder

In Terminal, type:

cd /Users/juanpaulomejia/projects/website

Then press Enter. You are now inside your website folder.

3 Initialize Git in your website folder

Run these commands one at a time, pressing Enter after each:

git init
git add .
git commit -m "Initial website upload"

This tells Git to start tracking your files and saves your first snapshot.

4 Connect your folder to GitHub

On the GitHub page you left open, copy the URL that looks like:

https://github.com/YOUR-USERNAME/professional-services.git

Then in Terminal run (replacing the URL with yours):

git remote add origin https://github.com/YOUR-USERNAME/professional-services.git
git branch -M main
git push -u origin main

GitHub may ask for your username and password — enter your GitHub credentials.

If it asks for a password and your normal password doesn't work, GitHub now requires a "Personal Access Token." Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Generate New Token (classic). Check the "repo" box and use the token as your password.
5 Enable GitHub Pages to make the site live

In your GitHub repository:

  • Click Settings (top tab)
  • Click Pages in the left sidebar
  • Under Branch, select main and click Save

Wait about 1–2 minutes, then your site will be live at:

https://YOUR-USERNAME.github.io/professional-services
Your website is now publicly accessible on the internet — for free!
6 How to update the site in the future

Whenever you make changes to your files, run these three commands in Terminal:

cd /Users/juanpaulomejia/projects/website
git add .
git commit -m "describe what you changed"
git push

The live site will update automatically within a minute or two.

Part 3 — Set Up Supabase to Store Contact Form Data Takes about 15 minutes

What is Supabase?

Supabase is a free cloud database. When someone fills out the contact form on your website, their information (name, phone, email, etc.) gets saved there. You can log in at any time to view all submissions in a table — like a spreadsheet in the cloud.

1 Create a Supabase account

Go to supabase.com and click Start your project.

Sign up using your GitHub account (easiest) or your email.

2 Create a new project

After logging in, click New Project.

  • Name: professional-services
  • Database Password: Create a strong password and save it somewhere safe
  • Region: West US (for California)

Click Create new project. Wait about 1 minute for it to finish.

3 Create the contacts table

In the left sidebar click SQL Editor, then click New Query.

Copy and paste the following, then click Run:

create table contacts (
  id uuid default gen_random_uuid() primary key,
  first_name text,
  last_name  text,
  email      text,
  phone      text,
  address    text,
  service    text,
  message    text,
  created_at timestamp default now()
);
You should see "Success. No rows returned." — that means the table was created.
4 Get your API credentials

In the left sidebar click Project Settings → API.

You will see two values you need:

  • Project URL — looks like https://abcdefgh.supabase.co
  • anon public key — a long string of letters and numbers

Copy both values and keep them ready for the next step.

5 Add your credentials to the website

Open the file /Users/juanpaulomejia/projects/website/script.js in VS Code.

Find these two lines at the top:

const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';

Replace YOUR_SUPABASE_URL with your Project URL and YOUR_SUPABASE_ANON_KEY with your anon key. Keep the quotes.

6 Set up Formspree for email notifications

Go to formspree.io → Sign up with paulo.j.mejia@gmail.com.

Click New Form, name it "Professional Services Contact", click Create Form.

Formspree will give you a URL like https://formspree.io/f/xpzgkwqr.

The part after /f/ is your Form ID (e.g. xpzgkwqr).

In script.js replace:

const FORMSPREE_ID = 'YOUR_FORMSPREE_ID';

With your actual Form ID.

7 Push the updated script.js to GitHub

After saving script.js, go to Terminal and run:

cd /Users/juanpaulomejia/projects/website
git add script.js
git commit -m "Add Supabase and Formspree credentials"
git push

Your live site will update within 1–2 minutes.

8 Test the form end to end

Visit your live site and submit a test contact form.

Then check:

  • Gmail — you should receive an email from Formspree with the submission details
  • Supabase → Table Editor → contacts — you should see a new row with the data
If both checks pass — everything is working! Your site is live and collecting data.

How to View Your Contacts in Supabase

Any time you want to see who has filled out the form:
  • Go to supabase.com and log in
  • Click your project → Table Editor in the left sidebar
  • Click the contacts table
  • All submissions are listed there with name, email, phone, service, message, and date
You can also filter, search, and export the data as a CSV file from this view.
