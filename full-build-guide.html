A complete step-by-step guide — from zero to live on the internet

GitHub Pages — Free Hosting Supabase — Database Formspree — Email Notifications No monthly cost to start

  1. Plan your website — pages, audience, services

Before You Start — Accounts You Need to Create

All three are free. Create them before following this guide.
1 GitHub — Where your website lives

GitHub stores your website files in the cloud and hosts your site for free using a feature called GitHub Pages. Without GitHub, your site only exists on your computer and no one else can see it.

  • Go to github.com and click Sign up
  • Create a free account using your email
  • Choose a professional username — it will appear in your site URL (e.g. yourname.github.io)
  • Verify your email address when prompted
Free. No credit card required.
2 Supabase — Where contact form data is stored

Supabase is a free cloud database. Every time someone fills out your contact form, their name, email, phone, and message gets saved here automatically. You can log in at any time to view all submissions — like a spreadsheet in the cloud.

  • Go to supabase.com and click Start your project
  • Sign up using your GitHub account (easiest) or your email
  • You will create a project and a contacts table during setup — this guide walks you through that step by step
Free tier available. No credit card required to get started.
3 Formspree — How you get notified of new submissions

Formspree sends you an email notification every time someone submits your contact form. Without it, you would have to manually log into Supabase to check for new messages. With Formspree, you get an instant email with all the details delivered to your inbox.

  • Go to formspree.io and click Get Started
  • Sign up using the email address where you want to receive notifications
  • You will create a form during setup — this guide walks you through it
Free tier includes up to 50 submissions per month. No credit card required.
4 VS Code — Where you write your website files

VS Code is a free text editor made by Microsoft. It's the tool you use to write and edit your HTML, CSS, and JavaScript files. Think of it like Microsoft Word, but for code.

  • Go to code.visualstudio.com and click Download for Mac
  • Unzip the downloaded file and drag VS Code to your Applications folder
Free. No account required.
Summary: GitHub hosts your site → Supabase stores your data → Formspree emails you when someone contacts you. All three work together automatically once set up.

Part 1 — Plan Your Website

Before writing any code, decide what your site needs
1 Decide your pages

For a simple services website, these pages work well:

  • Home — welcome message and overview of services
  • Service Page 1 — detailed description of your first service
  • Service Page 2 — detailed description of your second service
  • Service Page 3 — and so on
  • Contact — form for visitors to reach you
2 Decide what information to collect on the contact form

Common fields: First name, Last name, Email, Phone, Address, Service interested in, Message.

Only collect what you actually need. Each field you add is one more thing visitors have to fill out.
3 Choose a layout

A left sidebar navigation with main content on the right is clean and easy to expand later. You can add new pages to the sidebar at any time without redesigning the whole site.

Part 2 — Build Your Website Files

Create the HTML, CSS, and JavaScript files on your computer
1 Install VS Code

Go to code.visualstudio.com and download VS Code for Mac. Unzip it and drag it to your Applications folder.

2 Create a folder for your website

Create a folder called website inside your projects folder. This is where all your files will live.

/Users/yourname/projects/website/
3 Create these files inside the folder
website/
  index.html       — Home page
  service-1.html   — First service page
  service-2.html   — Second service page
  service-3.html   — Third service page
  contact.html    — Contact form page
  styles.css      — All the styling (colors, fonts, layout)
  script.js       — Handles form submission to Supabase + Formspree
4 Key things every HTML page needs

Every page shares the same left sidebar navigation. Copy the sidebar code into each HTML file and mark the current page with class="active" on its nav link.

The contact page must load the Supabase library and your script before the closing </body> tag:

<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
<script src="script.js"></script>
5 The script.js file has three placeholders to fill in later
const SUPABASE_URL     = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
const FORMSPREE_ID      = 'YOUR_FORMSPREE_ID';

Leave these as placeholders for now — you will fill them in after setting up Supabase and Formspree.

Important: name your Supabase client variable something other than supabase to avoid a conflict with the library. Use supabaseClient instead.

Part 3 — Set Up Supabase

Free cloud database to store contact form submissions
1 Create a Supabase account

Go to supabase.com → click Start your project → sign up with GitHub or email.

2 Create a new project
  • Click New Project
  • Name it (e.g. professional-services)
  • Create a database password and save it somewhere safe
  • Choose the closest region (West US for California)
  • Click Create new project — wait about 1 minute
3 Create the contacts table

In the left sidebar click SQL Editor → New Query. Paste this and click Run:

create table contacts (
  id         uuid default gen_random_uuid() primary key,
  first_name text,
  last_name  text,
  email      text,
  phone      text,
  address    text,
  service    text,
  message    text,
  created_at timestamp default now()
);

A popup will ask about Row Level Security — click Run and enable RLS.

You should see: "Success. No rows returned."
4 Add the insert policy

Run a second query to allow your contact form to save data:

create policy "Allow public to insert contacts"
on contacts
for insert
to anon
with check (true);
You should see: "Success. No rows returned."
5 Get your API credentials

Go to Project Settings → API. You need two values:

  • Project URL — looks like https://xxxxxx.supabase.co
  • Publishable key — starts with sb_publishable_...
Use the Publishable key (safe for browsers). Never use the Secret key in a website.

Copy both values — you will paste them into script.js shortly.

Part 4 — Set Up Formspree

Free service that emails you every time someone submits your contact form
1 Create a Formspree account

Go to formspree.io → click Get Started → sign up with your email.

2 Create a new form
  • Click the + button to create a form
  • Name it (e.g. Professional Services Contact)
  • Set the email to receive notifications
  • Click Create Form
3 Get your Form ID

Formspree will show you a URL like:

https://formspree.io/f/mpqelqrl

The part after /f/ is your Form ID — copy it.

Part 5 — Connect Your Credentials

Add your Supabase and Formspree details to script.js
1 Open script.js in VS Code

Find the three placeholder lines at the top and replace them with your actual values:

const SUPABASE_URL      = 'https://xxxxxx.supabase.co';
const SUPABASE_ANON_KEY = 'sb_publishable_xxxxxxxxxxxx';
const FORMSPREE_ID      = 'mpqelqrl';

Save the file with Cmd + S.

2 Make sure the variable name is supabaseClient

In script.js, the line that creates the client must use supabaseClient, not supabase:

const supabaseClient = window.supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

And all references to it later in the file should also say supabaseClient.

Part 6 — Install Git & Create a GitHub Account

One-time setup — only needs to be done once per computer
1 Check if Git is installed

Open Terminal (Cmd + Space → type "Terminal" → Enter) and run:

git --version

If you see a version number — Git is installed, skip to Step 3.

If you see "command not found" — continue to Step 2.

2 Install Git
xcode-select --install

Click Install in the popup and wait for it to finish.

3 Set your name and email in Git
git config --global user.name "Your Full Name"
git config --global user.email "your@email.com"
4 Create a GitHub account

Go to github.com → click Sign up → create a free account.

Choose a professional username — it will appear in your website URL (e.g. yourname.github.io).

Part 7 — Push Your Website to GitHub

Upload your files so GitHub can host them
1 Create a new repository on GitHub
  • Log into github.com → click + (top right) → New repository
  • Repository name: professional-services
  • Set to Public
  • Leave README, .gitignore, and license unchecked
  • Click Create repository
2 Initialize Git in your website folder

In Terminal, run these one at a time:

cd /Users/yourname/projects/website
git init
git add .
git commit -m "Initial website upload"
3 Connect to GitHub and push
git remote add origin https://github.com/YOUR-USERNAME/professional-services.git
git branch -M main
git push -u origin main

It will ask for your GitHub username and password. For the password, use a Personal Access Token — not your regular password.

4 How to create a Personal Access Token
  • GitHub → click profile picture → Settings
  • Scroll to bottom → Developer settings
  • Click Personal access tokens → Tokens (classic)
  • Click Generate new token (classic)
  • Give it a name (e.g. website), check the repo checkbox
  • Click Generate token and copy it immediately
Save the token somewhere safe — GitHub only shows it once. Use it as your password in Terminal.

Part 8 — Enable GitHub Pages

Make your website publicly accessible on the internet
1 Go to your repository settings

In your browser go to:

github.com/YOUR-USERNAME/professional-services/settings/pages
2 Enable Pages
  • Under Branch, click the dropdown that says None
  • Select main
  • Click Save

Wait 1–2 minutes. Your site will be live at:

https://YOUR-USERNAME.github.io/professional-services
Your website is now publicly accessible on the internet — for free!

Part 9 — Test Everything

Confirm the form is saving data and sending emails
1 Submit a test form
  • Visit your live site → click Contact Us
  • Fill out the form with your own info
  • Click Send Message
  • You should see a green "Thank you! Your message has been sent." banner
If the page was open before the latest push, do a hard refresh: Cmd + Shift + R
2 Check Supabase
  • Go to supabase.com → your project → Table Editor → contacts
  • You should see a new row with the data you submitted
3 Check your email

You should receive an email from Formspree with all the form details — name, email, phone, service selected, and message.

If all three pass — your website is fully operational!
4 Troubleshooting — if the form doesn't work
  • Press Cmd + Option + I on the contact page → click Console tab
  • Submit the form and look for red error messages
  • Common issue: "supabase already declared" — make sure your client variable is named supabaseClient not supabase
  • Common issue: form works locally but not live — you must push to GitHub first, the form won't work on a local file

Part 10 — How to Update the Site in the Future

Making changes after the site is live
1 Make your changes in VS Code

Open any file, make your edits, and save with Cmd + S.

2 Push the changes to GitHub

Open Terminal and run:

cd /Users/yourname/projects/website
git add .
git commit -m "describe what you changed"
git push

The live site will update automatically within 1–2 minutes.

After pushing, do a hard refresh in the browser (Cmd + Shift + R) to see your changes.
3 Adding a new page to the site
  • Create the new HTML file in your website folder
  • Add a link to it in the sidebar nav of every other page
  • Push to GitHub using the commands above
